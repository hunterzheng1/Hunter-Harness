# 对照表输出模板

## 追加位置

在原文档末尾追加，以 `---` 分隔线与原文内容分开。

## 整体结构

```markdown
---

## S/4 表字段与实体类对照表

### 表名：{S/4表名}
- **实体类名**：{实体类名}
- **实体类路径**：{相对路径}

| S/4 字段名 | 实体字段名 |
|-----------|-----------|
| {S4字段1} | {实体字段1} |
| {S4字段2} | {实体字段2} |

---

### 表名：{S/4表名}
- **实体类名**：{实体类名}
- **实体类路径**：{相对路径}

| S/4 字段名 | 实体字段名 |
|-----------|-----------|
| {S4字段1} | {实体字段1} |
```

> ⚠️ `{S/4表名}` 是步骤1提取的原始 SAP 表名（如 ACDOCA、BKPF），不是 API 返回的实体类名（如 UniversalJournal）。

## 格式规则

| 规则 | 说明 |
|------|------|
| 标题 | 使用 `##` 二级标题 "S/4 表字段与实体类对照表" |
| 表块标题 | `### 表名：{S/4表名}`，**必须是 S/4 原始表名（如 ACDOCA），严禁用实体类名（如 UniversalJournal）** |
| 实体信息 | 用无序列表展示实体类名和实体类路径 |
| 字段表格 | 两列表格：S/4 字段名 \| 实体字段名 |
| 表块分隔 | 不同表块之间用 `---` 分隔 |
| 排序规则 | 表按字母顺序排列，表内字段也按字母顺序排列 |
| 未找到标记 | 实体类名、路径、字段名未找到时统一标记为 "未找到" |
| T-table 合并 | 场景A：字段合并到基表块，表格增加"来源表"列；场景B：独立为块，标注基表实体类 |

## T-table 合并输出模板

### 场景 A：基表同时出现（字段合并）

```markdown
---

## S/4 表字段与实体类对照表

### 表名：{基表S/4表名}
- **实体类名**：{基表实体类名}
- **实体类路径**：{基表实体类路径}

| S/4 字段名 | 实体字段名 | 来源表 |
|-----------|-----------|--------|
| {S4字段1} | {实体字段1} | {基表名} |
| {S4字段2} | {实体字段2} | {基表名} |
| {S4字段3} | {实体字段3} | {T-table名} |
```

> 合并场景下表格增加"来源表"列，明确标注每个字段来自基表还是 T-table。

### 场景 B：T-table 单独出现

```markdown
---

## S/4 表字段与实体类对照表

### 表名：{T-table名}
- **实体类名**：{基表实体类名}（来自基表 {基表名}）
- **实体类路径**：{基表实体类路径}

| S/4 字段名 | 实体字段名 |
|-----------|-----------|
| {S4字段1} | {实体字段1} |
```

> 场景B中实体类名后标注"（来自基表 XXX）"，路径使用基表的实体类文件路径。

## 生成时间戳

在对照表标题下方添加生成时间：

```markdown
> 生成时间：YYYY-MM-DD HH:mm
```
